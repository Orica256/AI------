# プロジェクト全体ルール（ルートCLAUDE.md）

## プロジェクト名
**就活コンパス（Shukatsu Compass）** — 就職活動の応募先・選考状況・締切を一元管理するWebアプリ

## 目的
就活中の大学生が「どの企業がどの選考段階か」「次の締切・面接はいつか」を一目で把握できるようにする。
作者本人（大学3回生・就活開始直後）が実際に使え、かつポートフォリオとしても提示できることを狙う。

## 絶対制約
1. **完全無料 / ローカル完結**：有料クラウド・有料API・課金サービスは一切使わない。すべてオープンソース＆ローカル実行。
2. **個人情報を外部送信しない**：データはローカルのSQLiteファイルにのみ保存。
3. **誰でも `npm install` → 起動コマンドで動く**こと。

## 技術スタック
- フロントエンド：React + TypeScript + Vite
- バックエンド：Node.js + Express + better-sqlite3（SQLite）
- 通信：REST API（JSON）
- 配置：アプリ本体は `app/`（モノレポ）に集約。`app/backend/`（API）と `app/frontend/`（UI）。
  - 各担当フォルダ（`バックエンドエンジニア/` 等）には作業ログ・参考資料など**作業記録のみ**を残す。実際に動くコードは `app/` を正とする。

## チーム構成と責任
| 役割 | 主担当 | 成果物の置き場所 |
|------|--------|------------------|
| PM | 進捗管理・最終チェック・指示・承認 | `PM/成果物/` |
| フロントエンドエンジニア | UI実装・資料作成・PM報告 | コード：`app/frontend/` ／ 作業記録：`フロントエンドエンジニア/` |
| バックエンドエンジニア | API実装・資料作成・PM報告 | コード：`app/backend/` ／ 作業記録：`バックエンドエンジニア/` |
| レビュアー | 品質保証・PM承認確認 | `レビュアー/成果物/` |
| 秘書 | ユーザー意見の整理・PM共有 | `秘書/成果物/` |

## フォルダ共通ルール
- 各メンバーフォルダには `参考資料/`（調査・引用元）と `成果物/`（実際の作業物）を必ず持つ。
- 各メンバーは自分の `作業ログ.md` に作業内容を時系列で追記する。
- 全体の進捗は ルート直下の [`作業ログ.md`](作業ログ.md) から各メンバーログへ辿れる。

## ワークフロー（1サイクル）
1. **秘書** がユーザーの意見・要望を整理 → PMへ共有
2. **PM** が企画・要件・指示書を作成 → 各エンジニアへ作業指示
3. **バックエンド/フロントエンド** が実装 → PMへ報告
4. **レビュアー** が品質チェック（コードレビュー）→ 指摘 or 合格
5. **PM** が最終チェック・承認 → 次サイクルへ

## コーディング規約
- 言語：TypeScript（strict）。
- 命名：変数・関数は camelCase、型・コンポーネントは PascalCase。
- コメント：日本語可。意図が分かりにくい箇所のみ簡潔に。
- 1ファイル1責務を意識し、肥大化させない。
