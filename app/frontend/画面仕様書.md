# 画面仕様書（就活コンパス フロントエンド）

作成：フロントエンドエンジニア ／ 2026-06-12

## 起動方法
```bash
cd app/frontend
npm install
npm run dev      # http://localhost:5173
```
※ 先にバックエンド（localhost:3001）を起動しておくこと。
`/api` へのリクエストはViteのプロキシで自動的に3001へ転送される。

## 画面構成
ヘッダーのナビゲーションで3つのビューを切替（ルーター不使用・状態管理のみ）。

### 1. ダッシュボード（Dashboard.tsx）
- 選考状況サマリー：ステータス別の企業件数をグリッド表示。
- 今後7日以内の締切：企業名（クリックで詳細へ）・ステータス・締切日。
- 今後7日以内の予定：未完了イベントを企業名・タイトル・日付で表示。

### 2. 企業一覧（CompanyList.tsx）
- テーブル表示（企業名／業界／ステータス／志望度★／締切／予定件数）。
- ステータスでのフィルタ（チップ型ボタン）。
- 「＋企業を追加」でインラインの登録フォームを開閉。
- 行クリックで詳細へ遷移。

### 3. 企業詳細・編集（CompanyDetail.tsx）
- 企業情報の表示／編集（編集ボタンでフォーム化）。
- 削除（確認ダイアログ付き）。
- 選考イベントの一覧・追加・完了チェック・削除。

## 設計メモ
- API通信は `src/api.ts` に集約。エラーはメッセージを画面上部に赤背景で表示。
- 型は `src/types.ts` でバックエンドと対応（Status enum等）。
- 外部UIライブラリ不使用（軽量・無料の方針）。スタイルは `styles.css` のみ。

## 第4サイクル 追加（2026-06-13）
- **A1 検索ボックス**（企業一覧）：企業名・業界の部分一致でクライアント側フィルタ。ステータスフィルタと併用可。
- **A2 並べ替え**（企業一覧）：「デフォルト／締切が近い順／志望度が高い順」を選択。締切なしは末尾。
- **A3 締切リマインド強調**（ダッシュボード）：締切に `DeadlineBadge` を表示し、超過・当日・3日以内の件数を「🔥 緊急 N件」で見出しに強調。
- **A4 メモのMarkdown表示**（企業詳細）：メモを簡易Markdown（見出し`#`／箇条書き`-`／太字`**`／改行）で表示（`SimpleMarkdown.tsx`、無依存・XSS安全）。編集時はtextareaのまま。
- **A5 ダークモード**：ヘッダのトグルで切替。CSS変数（`[data-theme="dark"]`）で全体に適用し、`localStorage` で保持。

## 第5サイクル 追加・改善（2026-06-13）
- **改善1 ToDoの期日**：企業詳細のToDo追加フォームに任意の日付欄。期日ありは一覧に「期日 YYYY-MM-DD」を表示、未入力なら期日なし（`tasks.due_date`）。
- **改善2 かな検索**：企業一覧の検索でひらがな↔カタカナを正規化して一致（「てっく」で「テック…」がヒット）。
- **改善3 企業名ソート**：並び順に「企業名 昇順（あ→ん）／降順（ん→あ）」を追加（`localeCompare('ja')`）。
- **改善4 カンバンの横スクロール解消**：カンバンをグリッド化して画面幅に収め、カンバン表示時は `main` を `wide`（max-width 1280px）に拡張。

## 第6サイクル 追加・改善（2026-06-13）
- **改善5 カレンダーにToDo期日**：期日付きToDoを `✅` で該当日に表示（`GET /api/tasks`）。📌選考イベント・⏰締切と区別し、凡例も更新。
- **改善6 インターン関連ステータス**：選考ステータスに「インターン応募／インターン選考／インターン参加」を追加（計11段階）。一覧フィルタ・カンバン・ダッシュボードに反映。
- **カンバン列数の自動連動**：カンバンの列数を `STATUSES.length` に連動（インラインstyle）。ステータスが増減しても横スクロールせず1画面に収まる（改善4の固定8列を汎用化）。
