# Codex × Claude 連携案（コーディング外部委譲）

> ステータス：**フェーズ0–1 導入済み**（2026-06-12 実機確認・セットアップ完了）。フェーズ3（試運転で本採用判断）は未実施。
> 最終更新：2026-06-12

## 目的
コードの**生成（書く作業）が最もトークンを消費する**ため、そこを ChatGPT Pro の **Codex** に委譲し、Claude は「設計・レビュー・ドキュメント・進捗管理」に集中してトークンを節約する。

## 目指す形（VS Code ターミナル1枚で完結）
ユーザーは **Claude Code（1ターミナル）にだけ指示**する。
Claude が裏で：仕様を書く → `codex` を Bash 経由で起動して実装させる → `git diff` をレビュー → ドキュメント＆コミット。
→ Codex の画面を別で開く必要なし。コーディングのトークンは Codex（ChatGPT Pro）側へ。

## 役割分担
| 工程 | 担当 | 理由 |
|------|------|------|
| 要件・設計・**API契約／TypeScript型の確定** | Claude | 仕様の“正”を一元管理し、両者の衝突を防ぐ |
| 実装（`app/` のコード） | **Codex** | 生成トークンを Codex 側へ逃がす |
| レビュー（`git diff` を読む） | Claude | 読む＜書く で相対的に安い |
| ドキュメント・進捗・コミット文面 | Claude | テキスト主体で軽い |

ポイント：**インターフェース（APIの形・型）は Claude が先に固定**し、Codex は中身を埋めるだけにする。

## 運用ループ（1指示あたり）
1. Claude：要件 → 仕様＋型/API契約を確定（`app/` のインターフェースの正は Claude が握る）
2. Claude：実装を Codex に委譲（コードは Codex が書く）
3. Claude：`git diff` をレビュー → 必要なら 2 へ差し戻し
4. Claude：ドキュメント更新＋**段階ごとにコミット＆プッシュ**（author=Orica256・Claude表記なし）

## セットアップ作業案（後で実施）
**フェーズ0：前提確認（最重要）**
- Codex CLI をインストールし、ChatGPT Pro でログイン。
- **非対話（ヘッドレス）実行が可能か確認**：プロンプトを渡すと確認なしでファイルを編集して終了するモードがあるか。
  - 無い／毎回承認が要る仕様なら「1ターミナル自動運用」は不可 → 手動リレー（Claude が仕様出力 → 人が Codex に貼る）に格下げ。

- ハンドオフ用テンプレ：Claude が書く「作業指示（完了条件つき）＋API契約／型」を Codex がそのまま読める1枚に。

**フェーズ2：運用ループの固定化**（上記4ステップ）
- 検証（`tsc --noEmit`・起動・curl）は Codex 側に寄せ、Claude は結果ログのみ確認。

**フェーズ3：小さく試す**
- 1エンドポイント追加など極小タスクで1周し、トークンの出方・手戻り・競合を観察してから本採用。

## 決めておくこと
- 境界線：型/API契約は Claude 固定 → Codex は中身だけ、で良いか。
- レビュー深度：Claude は「差分だけ」か「起動確認まで」か。
- アクター：バック/フロントの“中の人”だけ Codex に置換（PM/レビュアー/秘書/ドキュメントは Claude）。
- コミット権限：**Claude 固定**を推奨（履歴の乱れ・著者統一のため）。

## リスク・確認事項
1. Codex CLI のヘッドレス対応可否（要実機確認）。NG なら手動リレー。
2. 同時編集の競合 → 必ず逐次（実装→レビュー）。並行させない。
3. 規約のズレ → Codex にも `CLAUDE.md`（無料/ローカル・命名規則）を毎回渡す。
4. レビューにも読みトークンは要る → 差分中心で最小化。

## 実機セットアップ記録（2026-06-12）
このPC（Windows 11 / PowerShell）で前提確認とフェーズ1の整備を実施。

### フェーズ0：前提確認（結果＝全クリア）
- ✅ **Codex CLI 導入**：`npm i -g @openai/codex` → `codex-cli 0.139.0`（node v24.13.1 / npm 11.8.0）。
- ✅ **ログイン済み**：`codex login status` → "Logged in using ChatGPT"。
- ✅ **ヘッドレス実行あり**：`codex exec`（非対話）が存在。`--sandbox read-only|workspace-write|danger-full-access`、
  `--cd <DIR>`、`-c model_reasoning_effort=...` 等を確認。`approval: never` で確認なし実行が可能。
- ✅ **疎通テスト成功**：read-only で最小プロンプトを投げ応答を取得（model = gpt-5.5）。
  → **手動リレーへの格下げは不要。1ターミナル自動運用が可能。**

### フェーズ1：呼び出し経路（整備済み）
- `scripts/codex-task.ps1` … ハンドオフ仕様ファイルを受け取り、規約ヘッダを添えて `codex exec` を起動するラッパー。
  既定は `--sandbox workspace-write`（リポジトリ内のみ書込）/ `model_reasoning_effort=medium`（トークン節約）。実行ログは `scripts/logs/` に保存。
- `scripts/handoff-template.md` … 1タスク=1ファイルの作業指示テンプレ（背景／対象／確定API契約・型／作業内容／完了条件／制約）。
- 規約のズレ防止（リスク3）：ラッパーが「無料・ローカル／型・API契約は変更不可／app/ にのみ実装／コミットしない」をプロンプト先頭に毎回付与。

### コスト注意
- `~/.codex/config.toml` の既定は `model_reasoning_effort = "high"`（疎通テストで約1.9万トークン消費）。
  ラッパーは既定 `medium` に下げ、難設計のみ `-Effort high` を渡す運用とする。

## フェーズ3：試運転の結果と方針確定（2026-06-13）
極小タスク（ドキュメントの起動パス修正）を `scripts/codex-task.ps1`（workspace-write / medium）で1周委譲し、計測した。

### 結果
- パス修正自体は正確。**ただし指示を逸脱**：
  1. 「テキスト置換」の指示に対し、対象3ファイル（README/API仕様書/画面仕様書）を**全文リライト**し、既存情報を一部欠落（メモ機能・status enum具体値・動作確認チェックリスト等）。
  2. **対象外の CLAUDE.md・作業ログ.md を全面改変**（チーム構成・ワークフロー・メンバー別ログ索引が消失）。
  3. **指示した 共有事項.md を取りこぼし**（旧パス残存）。
- レビューが無ければ気づけない改変だった＝レビュー工程は必須。

### トークン（実測）
- Codex（ChatGPT Pro側）：疎通 19,471(high) / スモーク 46,540(read-only,low) / 本タスク **123,147(write,medium)**。
- 同タスクを Claude 直接なら **1〜2万規模**・スコープ逸脱なし・取りこぼしなし（対象ファイルは既読のため編集のみ）。
- リポジトリ全読み込みの固定コストが大きく、小〜中タスクでは本体作業を委譲オーバーヘッドが上回る。

### 確定した運用方針
- **本採用は見送り。既定＝Claude、ユーザーが明示的に指示したときのみ Codex を使う**（[`CLAUDE.md`](CLAUDE.md) に明記）。
- 委譲する場合は必ず：API契約/型は Claude が確定 → ラッパー経由 → Claude が差分レビュー → Claude がコミット。
- 大量のコード新規生成タスクでは費用対効果が変わる可能性があり、その時点で再評価する（ラッパー・テンプレは整備済みのため即再開できる）。

## 既知の注意（共有の落とし穴）
- Claude Code の**メモリ（`~/.claude/...`）はPCローカルで、gitに入らない＝別PCに引き継がれない**。
  恒久ルールは本ファイルや [`CLAUDE.md`](CLAUDE.md)・[`共有事項.md`](共有事項.md) など**リポジトリ内**に書くこと。
