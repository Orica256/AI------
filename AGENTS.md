## プロジェクト
就活コンパス：応募先・選考状況・締切・ToDo を管理するローカル完結の Web アプリ。
- フロント：React + TypeScript + Vite（`app/frontend`）
- バック：Express + better-sqlite3（`app/backend`）

## 絶対制約（厳守）
1. **完全無料・ローカル完結**：有料 API / 有料クラウド / 課金サービスを使わない。個人情報を外部送信しない（データはローカル SQLite のみ）。
2. **動くコードは `app/` 配下にのみ書く**（`app/backend`・`app/frontend`）。`PM/`・`秘書/`・`レビュアー/`・各種 `*.md`（チームドキュメント）や `CLAUDE.md`・`作業ログ.md` は**変更しない**。
3. **git commit / git push はしない**。コミットは人間（Claude）が author=`Orica256 <hatopal1001@gmail.com>` で行う。

## 型・API 契約（最重要）
- API のパス・レスポンス形・TypeScript 型・DB スキーマの「正」は、**作業指示（ハンドオフ）で与えられる**。それに厳密に従い、**勝手に変更しない**。
- 変更が必要だと判断したら、実装せず理由を出力して止まる。
- ステータスの enum は `app/frontend/src/types.ts` と `app/backend/src/db.ts` の**両方**にあり、**片側だけ変更しない**。

## 作業マナー（スコープ厳守）
- **指示された範囲だけ**を変更する。既存ファイルの無関係な箇所・文体・見出し構成を勝手に書き換えない。
- **「全文リライト」をしない**。必要な追加・修正を**最小差分**で行う（既存の良い記述・情報を消さない）。
- 依存パッケージの**無断追加をしない**（必要なら理由を述べて止まる）。
- DB：新規テーブルは `CREATE TABLE IF NOT EXISTS`。既存テーブルへの列追加は**冪等マイグレーション**（`PRAGMA table_info` → 無ければ `ALTER TABLE`）で行い、既存データを壊さない。
- 既存のスタイル・命名・周辺コードの書き方に合わせる。
