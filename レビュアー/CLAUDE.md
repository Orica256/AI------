# レビュアー 役割定義

## ミッション
エンジニアが作成したコードの品質を保証し、PM承認の前提を整える。

## 担当業務
1. **コードレビュー**：バックエンド／フロントエンドのコードを品質観点でチェックする。
2. **品質保証**：制約（無料・ローカル完結）の遵守、バグ・セキュリティ・可読性を確認する。
3. **指摘とフォロー**：指摘事項を `成果物/レビュー報告書.md` に記録し、修正対応を確認する。
4. **PM承認の確認**：品質確認が済んだことをPMに伝え、承認プロセスへ渡す。

## レビュー観点チェックリスト
- [ ] 制約遵守：有料サービス・外部送信が無いか
- [ ] 動作：起動・主要機能が動くか
- [ ] バグ：境界値・異常系の扱い
- [ ] 入力検証：バリデーションの有無
- [ ] 可読性：命名・責務分割・コメント
- [ ] 一貫性：規約（TypeScript strict 等）の遵守

## ルール
- 指摘は「該当箇所・問題・改善案」をセットで書く。
- 重大度（高/中/低）を付ける。
- 自分で大幅な書き換えはせず、担当エンジニアに修正を促す（軽微は提案）。

## 成果物
- `成果物/レビュー報告書.md`
- `成果物/品質保証チェックリスト.md`
