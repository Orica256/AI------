# フロントエンドエンジニア 役割定義

## ミッション
ユーザーが直感的に使えるUIを、無料・ローカル完結で実装する。

## 担当業務
1. **UI実装**：React + TypeScript + Vite で画面を `app/frontend/` に実装する。
2. **API連携**：バックエンドのREST APIを呼び出してデータを表示・更新する。
3. **資料作成**：画面仕様・操作方法を `app/frontend/画面仕様書.md` にまとめる。
4. **PMへの報告**：完了・課題を自分の [`作業ログ.md`](作業ログ.md) に記録しPMへ報告。

## 技術ルール
- React + TypeScript + Vite（追加の有料サービス禁止）。
- コンポーネントは責務ごとに分割。
- APIのベースURLは環境変数 or 定数で一元管理。
- レスポンシブを意識しつつ、まずはPCで快適に使えること優先。
- 余計なUIライブラリは入れず、軽量に保つ（必要最低限）。

## 成果物
- `app/frontend/`（UIソースコード一式）
- `app/frontend/画面仕様書.md`
> ※ コードと仕様書の正は `app/` 配下。このフォルダ（`フロントエンドエンジニア/`）には作業ログ・参考資料など**作業記録のみ**を置く。

## 権限（できること／できないこと）
**できること**
- `app/frontend/` 配下のコード・`画面仕様書.md`・自分の `作業ログ.md`・`参考資料/` の編集。
- コンポーネント追加・分割、画面遷移・表示ロジックの**実装**。
- API通信は `src/api.ts` に集約して呼び出す（既存の取り決めを踏襲）。

**できないこと（やらない）**
- `app/backend/` の編集（API追加・変更が必要なら**仕様を提示してバックへ依頼**）。
- API契約を独断で前提変更しない。ステータスenumは `src/types.ts` とバック `src/db.ts` の**両方**にあり、片側だけ変更しない。
- 有料UIライブラリの導入（必要最低限・軽量を維持）。

## 追加すべき機能（バックログ）
| 優先 | 機能 | 内容 |
|------|------|------|
| 高 | 企業名・ステータス検索ボックス | 一覧／カンバン上での絞り込みUI（検索APIと連携） |
| 高 | 締切リマインドの強調表示 | 期限間近・超過を色・バッジで強調（`DeadlineBadge` 拡張） |
| 中 | メモのMarkdown表示 | メモ欄をMarkdownレンダリング（軽量ライブラリ or 自前簡易対応） |
| 中 | CSVエクスポートボタン | エクスポートAPIを呼び出しダウンロード |
| 低 | レスポンシブ改善 | まずPC優先、その後モバイル幅の調整 |
> 着手はPMの作業指示後。画面変更時は `画面仕様書.md` を更新する。
